## 2026-03-31 – JWT Authentication Planning

### Decision
Will implement JWT-based authentication instead of session/cookie auth.

### Reasoning
- Keeps API stateless
- Aligns with modern SPA/mobile patterns
- Easier to scale later

### Tradeoffs
- Token revocation is harder
- Slightly more complexity vs cookies

### Notes
Will keep implementation simple (no refresh tokens yet).
Focus is learning + demonstrating core auth flow.
